    Set up shelf check locations
    In This Topic

    First specify which locations are to be used with shelf check:

    1. From the Module Menu select Maintenance, Circulation and then Locations.
    2. Select the location you want from the list.
    3. On the Properties tab, set Shelf check location to Yes.
    4. Repeat steps 2 and 3 for every shelf check location.
    5. Click Save.
    6. Shut down the application and then restart to refresh the location parameters with your changes.

    Once you've specified which locations are to be used with shelf check, you can set up the tiers:

    1. From the Module Menu select Maintenance, Circulation and then Locations.
    2. Select the location you want from the list.
    3. Click the Shelf Check Locations tab.
    4. Click New to add a new tier. You can use Move up or Move Down to change the position of a tier. There is no limit to the number of tiers you can add.
    5. Set the Elapsed days or Elapsed minutes. This is the number of days or minutes since the reservation was placed. Once this period has passed, Spydus will search for an item in that tier. For example, for tier 1 you might set Elapsed days/minutes to 0 so that you start searching straight away, whereas for tier 2 you might set Elapsed days to 2 or 3.
    6. Select the Institution, and then click Add to add the locations to the selected tier. For example, tier 1 might have the locations that are nearest the pickup location, tier 2 might have the locations that are further from the pickup location, and tier 3 might have the locations that are furthest from the pickup location.
    7. Repeat steps 2 to 6 for each shelf check location.
    8. Click Save or Save & Close.

    You'll see an institution in the dropdown list at step 6 if it has any shelf check locations, i.e. if it has any locations with Shelf check location set to Yes. Global reservations active does not need to be set to Yes, and shelf check does not need to be active for that institution. This allows you to set up the parameters before turning shelf check on.

    The server code that processes the shelf check tiers and locations has checks to make sure it only assigns a reservation to a location from another institution if that institution has Global reservations active set to Yes.
