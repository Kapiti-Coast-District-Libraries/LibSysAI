    Cash drawer parameters
    In This Topic

    Cash drawer parameters control whether cash drawers are used for transactions, what user validation is required to process transactions, and whether a user will be prompted to close a cash drawer on logout. To set cash drawer parameters;

    1. Navigate to Maintenance > Circulation > General Parameters.
    2. In the Cash Drawer section, set parameters as required:                

    Use cash drawer

    This parameter controls whether cash drawers are being used and - if in use - whether it is optional or mandatory to use cash drawers.

    • No - Cash drawers will never be used for payments
    • Yes - Cash drawers may be used for payments
    • Yes, Mandatory - Cash drawers must be used for payments

    Additional staff credentials not required    

    This feature requires server version 10.9 or higher.  

    This parameter controls whether to prompt for staff credentials when connecting to, kicking, editing or closing a cash drawer. Prompting for staff credentials provides an additional layer of security when accessing cash.

    • No - When connecting to, kicking, editing or closing a cash draw, Spydus will prompt staff to validate their credentials                 
    • Yes -  When connecting to, kicking, editing or closing a cash draw, Spydus will not prompt for staff credentials
    This functionality is intended for use with single sign-on, where 2-factor authentication for SSO accounts while using cash drawers can be cumbersome. If users access Spydus without SSO, it is recommended to set this parameter to No (or null).

    Enable cash drawer options for generic staff

    This parameter controls whether cash drawer menu options are enabled when logged in staff member is a generic user.

    Generic users may not process cash transactions, and will be required to provide valid (non-generic) credentials to do so.

    • No - Normal staff privilege validation is applied e.g. If a generic user does not have sufficient privilege to view cash drawer options, they will not be displayed. If a generic user has sufficient privilege the cash drawer options will be displayed, though the options will require valid credentials to perform their actions.
    • Yes - If set to yes, a generic user account that otherwise does not have sufficient privilege to view the cash drawer options will be able to see the options. As expected, the options will require valid credentials to perform their actions.

    Prompt whether to close cash drawer on logout

    This parameter controls whether (when there is an open cash drawer) a user who is logging out will be prompted to close the cash drawer before the client is logged out.

    • No - When logging out, the user will not be prompted to close the drawer, and the open cash drawer session will remain open.
    • Yes - When logging out, the user will be prompted to close the drawer. The user may opt to close the drawer, or to leave it open.
    1. Click Save or Save & Close.
    Null means the same as No.
