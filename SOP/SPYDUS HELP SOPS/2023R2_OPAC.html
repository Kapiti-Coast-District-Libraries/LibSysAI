    Spydus OPAC features in 2023R2
    In This Topic

     Support multiple online registration forms

    This feature requires license version 11.3 or later.  

    Spydus now supports the ability to offer multiple online registration forms at the OPAC.

    By default a single form will be provided, but if additional forms are created in Register Online Parameters then users at the OPAC will be presented with a page where the appropriate registration form may be selected.

    images/2023R2_DISCOVERY_JOINSEL_thumb.png

    Click image to enlarge

     Each form may have different fields displayed, different fields as mandatory, different category and/or location defaults, and more.

    See 2023R2 Maintenance New Features for more information.

    Add PDF viewer to OPAC

    This feature requires server version 10.9 or higher.  

    A PDF viewer has been added to allow borrowers to View PDF files.

    If the Digital Asset flag to Allow download from OPAC has been checked, the PDF viewer will also provide options to Print and to Download the file.

    Archival Requests

    Archival Requests may now be placed at the OPAC.

    images/2023R2_OPAC_ARC_RQ_thumb.png

    Click image to enlarge

    In order for a borrower to place an archival request:

    • an archive record visible at the discovery platform must be linked to at least one issuable item
    • at least one location must be configured as a Pickup location (archive request)

    When placing the request, a borrower may select:

    • the item being requested (if the archive record is linked to multiple issuable items)
    • the date to pickup the item
    • the pickup location
    images/2023R2_OPAC_ARC_RQ_PLC_thumb.png

    Click image to enlarge

    To find archival requests in order to process them, it is recommended to create a widget based on an Archival Request search that suits the library's workflow(e.g. where Status = Pending).  

    For more detail, please see the Processing archival requests article for more information on editing, moving, receiving & issuing archival requests.

    Show reservations of dependents on the dashboard for a head of family

    This feature requires license version 11.3 or later.  

     

    Reservations for dependent borrowers will now show on the Family account dashboard for the head of family. This includes:

    • Reservations available for pickup
    • Reservations not yet available
    • Standing reservations
    • Previous reservations (from the History section)
    This feature was introduced after being requested by Jean at Birmingham City on the Spydus Ideas Portal (SPY-I-741). This idea received 38 votes!

    Automatically remove access violation trap when borrower password is reset

    This feature requires license version 11.3 or later.  

    When a borrower's password is changed by any method (e.g. email reset, library staff in Circulation, etc), the access violation trap (trap code 'E') will now be automatically removed. 

    This feature was introduced after being requested by Jamie at Joondalup on the Spydus Ideas Portal (SPY-I-952). This idea received 47 votes!

    Display rolling 7 days of opening hours

    This feature requires license version 11.3 or later.  

    Libraries now have the option to display the branch opening hours for the next 7 days. These may be displayed alongside the regular opening hours, or instead of the regular opening hours. To enable the display of the next 7 days opening hours;

    1. Navigate to Maintenance > OPAC & Enquiry > Display Parameters
    2. Set the View dropdown to 'PAC', the Format dropdown to 'Location', and the Type dropdown to 'Full Display'.
    3. Set the Opening hours (current week) as required.
    4. Save or Save & Close.

    If there are customised hours displayed, regular hours will be displayed in strikethrough text. This allows users at the OPAC to discern how the upcoming week may differ from regularly scheduled hours of operation.

      Image & Masonry container types

    This feature requires license version 11.3 or later.  

    Two new container types have been implemented; Image list & Masonry list. Both container types allow image cards with descriptive text shown either below the image (on a static card), or on the reverse of the image (which will flip to display the text on mouseover or at a nominated interval).

    For detail on configuration of the new container types, see 2023R2 Maintenance features.
