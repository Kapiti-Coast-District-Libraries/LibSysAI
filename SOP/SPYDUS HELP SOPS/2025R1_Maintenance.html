    Support app-based multi-factor authentication at OPAC

    To support additional & more secure methods of multi-factor authentication for borrowers, a new parameter has been added to the OPAC section of Circulation General Parameters. 

    To allow borrowers to enable MFA via authenticator app, ensure that the 2FA - Enable use of authentication app for login parameter is set to Yes.

    For more information on the configuration of other supported MFA methods and the related communications, see the 2FA parameters for OPAC login page.

    Restrict OPAC account renewal by borrower category 

    Parameters have been added to control the ability of borrowers from specific categories to renew their expiring account at the OPAC.

    Allow renewal in OPAC

    In the General section on the Details tab of Borrower Categories Maintenance, the Allow renewal in OPAC parameter has been added.

    When null or set to Yes, borrowers belonging to the selected category will be allowed to renew their account at the OPAC when registration has expired or is approaching expiry (within the Registration expiry cutoff period).

    If set to No, borrowers belonging to the selected category may not renew their account at the OPAC.

    The default setting for this parameter is null (allowing renewal).

    Allow staff to configure OPAC renewal messages

    The My account tab of the OPAC HTML Configurator already allows staff to configure messages for borrowers that are eligible to renew their accounts at the OPAC. This new feature allows staff to display custom messages for borrowers whose registration has expired or is approaching expiry (within the Registration expiry cutoff period).

    Account due to expire message when OPAC renewal is not allowed

    This message will be displayed at the borrower dashboard & personal details page when a borrower's registration is approaching expiry.

    Account has expired personal details message when OPAC renewal is not allowed

    This message will be displayed at the borrower dashboard & personal details page when a borrower's registration has expired.

    Remember to enable the Display Parameters for Renewal/expired registrations. These parameters are found under View code OPAC > ID code BRW > Type code FULL_PD. Ensure that the following parameters have Enabled set to Yes.

    • Renew expired registration
    • Renew due to expire registration
    • Registration expired
    • Registration due to expire

    Add 'Icon' field to 'New Titles' & 'Previously Borrowed' sections in OPAC HTML Configurator

    Library staff can configure the OPAC to include icons on the cover image of 'New Titles' and 'Previously Borrowed' materials.

    This can be configured in Maintenance > OPAC & Enquiry > OPAC HTML Configurator > General tab, in the New Titles or Previously Borrowed sections. 

    To display both text & icon, ensure that both the 'Visual indicator text' and 'Icon' values have been set.

    To suppress the display of either the text or icon, remove the text or icon selection from the field.

    If New Title and/or Previously Borrowed text indicators have already been enabled, the system default icon selections will display when the 2025R1 release is deployed to your system.

    Add ability to maintain Bibliographic and Article collection filters in Enquiry

    It is now possible to maintain the available Filters when performing Bibliographic or Article searches in Enquiry.

    images/2025R1_MAINT_ENQ_AVAIL_FILTERS_thumb.png

    On the Search tab of the Enquiry HTML Configurator, in the Bibliographic and Articles sections, there is a table that is labelled Available filters. Adding a Collection filter to this table (and ensuring that this filter is applied to at least one Collection) will allow staff users to select that filter to perform searches.

    For more information, see the Collection filters page.  

    To add, edit, or delete the filters available to be selected in the Bibliographic & Articles tables, see the KCH table in Table Manager. 

     Option to display installed language packs in OPAC footer

    A new section Languages has been added to the Footer tab of the OPAC HTML Configurator to allow OPAC visitors to switch between installed language packs.

    By default, all installed language packs except for the default will be displayed in the table.

    If the Display language selector option is enabled, a table will be revealed that allows staff to choose which installed language packs will be displayed for selection by OPAC visitors. The Language pop-up menu will allow the visitor to select any of the installed languages for OPAC display (other than the currently selected language).

    Circulation Receipt & Acquisition Invoice integration with Authority

    Spydus now supports the export of Circulation Receipts & Acquisitions Invoices to the Authority local government management platform via a scheduled task.

    To configure Authority integration, navigate to Maintenance > General > Third-Party Settings and expand the Authority section.

    To reveal configuration options, staff must first set the Enable Authority Integration flag. Once enabled, all of the revealed fields are mandatory.

    Client ID and Client Secret fields will only display and apply if the Altitude application is selected.

    'Export from' dates

    Set the date from which the export file will be generated. Once successfully submitted to Authority, both invoices and receipts will be flagged as exported. The Export Batch field in the record will contain the ID of the batch, with the date & timestamp that the export task was completed.

    Once a record has been exported, the scheduled export task will ignore it in future exports regardless of the Export from date.

    e.g. The first run of the export on 01/03/2025 uses Export from date 01/01/2025. In batch 001, all invoices from 01/01/2025 - 01/03/2025 are exported and flagged as successfully submitted to Authority. On 02/03/2025, the scheduled task runs again. As all invoices from 01/01 - 03/01 have already been exported, only invoices from 01/03 - 02/03 will be exported.

    Vendor configuration for Invoice export

    To configure Vendor details for the Authority export, navigate to Maintenance > Acquisitions > Vendors > load the vendor > Authority tab.

    Scheduled jobs

    An interface has been built to give library staff visibility on the outcome of scheduled jobs configured by Civica staff. This feature is disabled by default and must be switched on by Civica staff.

    Currently, the only tasks that will be visible in this interface are:

    • Authority Invoice Export
    • Authority Receipt Export

    Library staff will only have View privilege on jobs that have been scheduled. Visible information includes the Trigger days/times of the schedule, the Last & Next run times, and the Last run result.

    Please contact Civica Support for assistance as commissioning is required.
