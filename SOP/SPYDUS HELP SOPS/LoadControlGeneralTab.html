    Load controls: General tab
    In This Topic

    Tags not wanted

    Click Tags not wanted to select the tags that you do not want to be loaded.

    • To add a tag to the list, type the tag in the top box and then click Add.
    • To delete a tag from the list, select the tag and then click Delete.

    Load all records as new (no matching performed) 

    Select this option if you want to load all new records without matching against existing ones. You might select this option when you know that none of the records you're about to load are on the database.

    Match incoming records

    Select this option if you want to match the new records against existing ones.

    Click Select matching tags to select the tags to be used for matching new records to existing ones. You can choose to match on the 001 tag or to match on 020, 022, 024, 028, 035 and 984. See Matching on ISBNs and Matching on specific prefixes for tag 035 below for more information.

    Select Load non-matching records to load records that do not match existing ones.

    Select Load matching records to load new records that match existing ones. i.e. when you want to bump records. Then do one of the following:

    • Select Replace existing record entirely. If the incoming record matches an existing one then the existing one will be replaced. The flags, filters and holdings from the existing record will be retained.
    • Select Retain existing record but load some incoming tags. Click Tags wanted to specify the tags you want to load.

    For non-repeatable tags; If the existing record already has the same tag then we DON’T load the one from the incoming record, i.e., we keep the existing tag.

    For repeatable tags; We keep the existing tag(s) and load the new one(s).

    • Select Replace existing record but retain some tags. For example, you might choose this option if you have site-specific tags in your records. Click Tags wanted to specify the tags you want to retain.

    For non-repeatable tags; If the incoming record also has the same tag then we replace the existing one with the new one.

    For repeatable tags; We keep the existing tag(s) and load the new one(s).

    • Select Retain existing record but load incoming tags if it does not exist. Click Tags wanted to specify the tags you want to load if they're not already in the existing record.

    Clear the Load records with lower encoding level option and the new record will be loaded only if its encoding level is equal to or higher than the existing record. This is the default setting. Select this option and the new record will be loaded even if its encoding level is less than the existing record. See Encoding levels for bibliographic and authority records below for more information.

    If you select Pick the best one if multiple matches found then Spydus will automatically pick the “best” one if it finds multiple matches. The "best" one is either the one with the highest encoding level or, failing that, the largest one (i.e. the one with the most tags). If you do not select this option and multiple matches are found then the record will not be loaded. You'll see a message:

    8 – matching performed, multiple matches found, did not load. Please choose to load with interactive load.

    If selecting the Use bibliographic fields where no direct match found option, if other matching criteria (see above) are not met, Spydus will attempt to match on bibliographic criteria in the MARC record.

    You can then try loading with the Use interactive load option so you can manually choose the record you want.

    Save control number in 001 to 035

    When loading a MARC file into Spydus, the existing 001 tag will be replaced with the BRN of the record. Using the Save control number in 001 to 035 option, the existing control number can be copied into the 035 tag with a chosen prefix to designate the record vendor e.g. (OcoLC) or ANBD

    This will allow batches of external MARC files to be uploaded, retaining the external system control number in the 035 tag.

    Matching on ISBNs

    When you match on ISBNs you only match on digits. You do not match on, for example, anything in parentheses. For example, if a record has 123456789X (pbk) in the 020 field, we try to match on 123456789X.

    Matching on specific prefixes for tag 035

    The 035 tag contains the system control number of a system other than the one whose control number is contained in tag 001. It consists of the organisation code in parentheses followed immediately by the control number of the other system, e.g. (OCoLC)1234567.

    The 035 tag is repeatable, so there may be many of these tags in the record with control numbers from many different systems.

    You can set up the MARC load control to specify which organisation codes to use for matching. Only tags with the specified codes (in parentheses) are used for matching. The codes must match exactly for punctuation and case.

    If no codes are specified then all 035 tags will be used for matching.

    Select the 035 in the Select matching tags dialog, then click Control ID to Match. Enter the organisation code required and click OK.

    Encoding levels for bibliographic and authority records

    The encoding level for bibliographic records is specified in character position 17 of the LDR tag. The encoding levels for bibliographic records, highest to lowest, are:

    # - Full level.

    4 - Core level.

    1 - Full level, material not examined.

    2 - Less-than-full level, material not examined.

    8 - Prepublication.

    7 - Minimal level.

    5 - Partial (preliminary) level.

    3 - Abbreviated level.

    The encoding levels for authority records, highest to lowest:

    n - Complete.

    0 - Incomplete.

    Bibliographic records created in Acquisitions are assigned an encoding level of 5 by default.

    Use bibliographic fields where no direct match found

    If this option is selected, and no other matching criteria have been met, Spydus will attempt to match based on the bibliographic details in the MARC record. The following tags and subfields will attempt to be matched:

    Tag Subfield(s)
    245 #a and #b
    100, 110, 111 #a
    000/06 Record type
    008/23 Form
    008/35/37 Language

    To be considered a match, all of the above fields must match. If a match is found, the record will be loaded based on the Load matching records settings.

    If no match is found, the record will be loaded only if Load non-matching records is checked.
