    Spydus Circulation features in 2023R2
    In This Topic

    Option to not require additional staff credentials on cash drawer actions

    This feature requires server version 10.9 or higher.  

    A new parameter has been introduced to allow users logging into Spydus using single-sign on to not require additional validation of staff credentials when performing cash drawer actions.

    The Additional staff credentials not required parameter has been added to the Cash Drawer section of Circulation General Parameters.

    If this parameter is set to Yes, then a user who:

    • has sufficient privilege, and
    • is not a generic user

    ...may process cash drawer actions without requiring the user to re-validate their credentials.

    Allow additional control over what fields are inherited between family members

    This feature requires license version 11.3 or later.  

    New controls on the Settings tab of Borrower Registration Parameters allow library staff to configure which details are inherited by default from a Head of family to a dependant.

    Additionally, once a dependant has been linked to a head of family, the Fields inherited by dependants section will no longer be displayed in the Borrower Registration of the dependant.

    This feature was introduced after being requested by Paul at Norforlk Libraries on the Spydus Ideas Portal (SPY-I-229). This idea received 16 votes!

    Verify notice delivery

    This feature requires license version 11.3 or later.  

    A new parameter has been added to the Settings tab of Borrower Registration Parameters that (if enabled) will check if a borrower has opted to receive notices via email or SMS if they have provided the relevant contact details, and prompt the user to select a delivery method.

    The prompt will only be displayed if the borrower's email address or the borrower's mobile number (or both) is being updated & saved. If a borrower currently has an email address or mobile number registered, but there is no change made to the borrower's registration, no prompt will be displayed.

    The email prompt will be displayed if:

    • the borrower has an E-mail address (or guarantor/employer email address) is set or updated
    • Send email flag is not set

    The SMS prompt will be displayed if:

    • the borrower Mobile number is set or updated
    • SMS notice delivery is not set
    This feature was introduced after being requested by Paul at Norfolk Libraries on the Spydus Ideas Portal (SPY-I-850). This idea received 26 votes!

     Option to clear guarantor details when a borrower account is moved to the succeeding category

    This feature requires license version 11.3 or later.  

     If the Clear guarantor details on move to succeeding category Borrower Categories parameter is set to Yes, when a borrower is moved to the succeeding category, all guarantor fields will be cleared from the borrower record.

    This feature was introduced after being requested by Sunshine Coast on the Spydus Ideas Portal (SPY-I-790). This idea received 30 votes!
