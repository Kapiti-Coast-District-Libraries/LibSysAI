    Spydus OPAC features in 2023R3
    In This Topic

    Allow a borrower within required retention period to request anonymisation

    This feature requires server version 10.9 or higher.  

    To comply with data retention regulations, libraries may set a period during which borrowers may not request to be forgotten. Previously, resigning a borrower with activity during the retention period would not present staff the option to request to be forgotten when resigning.

    This functionality has now been updated to allow borrowers to request to be forgotten within this period. Spydus will present an alert if a request is made to anonymise a borrower within the retention period, advising that the anonymisation request will not be processed until the date that the retention period elapses. As long as the borrower does not reactivate their account during this time, anonymisation will be completed automatically by the borrower anonymisation scheduled task at the appropriate time.

    This functionality also appliess to borrowers who resign via Circulation module.

    Amend base URL used for Addressify API endpoints at the OPAC

    At the OPAC, the base URL for Addressify API endpoints has been updated from:

    • https://addressifyapiau.azurewebsites.net/

    ..to the preferred:

    • https://api.addressify.com.au/
    Unlike the update of Addressify endpoints in the Circulation module, this change is not featured in Spydus 10 versions. 

     Allow manual address entry when Addressify is enabled

    If the to Enable manual entry of address option is checked in the Join online/Update personal details section of the OPAC HTML Configurator, borrowers registering online may enter their address manually if it cannot be found by Addressify (e.g. poorly mapped areas or new property developments).

    'Access Resource' and 'Display image' links made more prominent

    This feature requires license version 11.3 or later.  

    Changes have been made to the full display of online resources and digital assets at the OPAC to emphasise the most likely next action by the patron: Access resource and View full image.

     Option to extend header, body and footer to full screen width

    On some larger displays, OPAC content can appear centred with empty space on either side. If configured on the General tab of the OPAC HTML Configurator, the header, body, and footer of the OPAC may each now be extended to take up the full screen width.
