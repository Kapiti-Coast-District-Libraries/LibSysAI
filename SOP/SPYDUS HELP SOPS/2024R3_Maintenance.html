     Re-order Tab property fields (Containers)

    In Tab properties in previous versions, the Type drop-down was displayed beside the Zone dropdown, and the Hide heading was placed beside the Display option. To make the interface more contextually relevant, the Type drop-down now appears at the top of Tab properties (beneath the Enabled checkbox), and the Hide heading checkbox has been moved next to the Heading textbox.

     Work level loan history validation

    This feature was introduced after being requested by Albany Public Library on the Spydus Ideas Portal (SPY-I-1401). This idea received 33 votes!

    Previously when using the Check loan history functionality in Spydus, the system checked loan history based on the IRN of the bibliographic record. This would mean that after loaning one instance of a work (e.g. an audiobook), issuing a different instance of the same work (e.g. a paperback) would not trigger the loan history alert (LOANHIST).

    A new Circulation General Parameter has been introduced to optionally expand loan history validation to the work level, where all instances of the work that have been previously loaned by the borrower will be included when performing the check.

    Additionally, to ensure clarIty of the parameter functionality, the Check loan history parameter has been renamed to Check loan history on issue. This parameter must be set to Yes to enable work level validation.

    To set checking loan history validating at work level:

    1. Navigate to Maintenance > Circulation > General Parameters
    2. Scroll to the Loans section, and ensure that Check loan history on issue is set to Yes
    3. Set Apply loan history validation at the bibliographic work level to Yes
    4. Click Save or Save & Close.

    Once saved, loan history validation at issue will be expanded to the work level. When issuing a title, loan history will validate the history of all instances that have been loaned previously. If Apply loan history validation at the bibliographic work level is set to No, loan history validation will only include currently issuing instance history.

    Ensure that loan history validation is correctly configured to use this feature. There are multiple parameters that control this functionality, including at the borrower category level, Circulation General Parameters, and the borrower's registration details. Refer to the Check loan history page for more details.

     Predefined values in Borrower Join Online forms

    It is now possible to set pre-defined values in the fields of borrower registration forms at the OPAC, and to have fields with pre-defined values hidden from the registration forms. This feature accomodates changes previously made to XSL files by library staff to allow hidden form values to be set.

    To set preset values and hide borrower registration fields:

    1. Navigate to Maintenance > OPAC & Enquiry > Borrower Join Online
    2. Select the required View (OPAC_WP) and Type
    3. On the Form tab, select the required field
    4. Set the required value
      • For text fields, type in the required Preset value
      • For dropdown fields, select the required value from the Preset value dropdown
      • For multi-select fields
      • For checkbox fields, set the Checked by default option
    5. If required, set the Hidden checkbox to suppress the display of this field from the selected form Type
    6. Click Save or Save & Close.

     Custom welcome emails for borrower categories

    This feature requires license version 11.7 or later.  

    Previously, the same welcome email was sent to all borrowers regardless of category. Welcome email content can now be customised for borrower categories, with different email content for borrowers in a category based on whether they self-register online or are registered by staff.

    This feature was introduced after being requested by Kildare Libraries on the Spydus Ideas Portal (SPY-I-1244). This idea received 60 votes!

    A new Welcome email section has been added to the Notice Settings tab of Borrower Categories Maintenance, allowing staff to create personalised email content for each borrower category.

    The Content section will be delivered to borrowers registered into this category in the Circulation borrower registration process.

    The Join online content section will delivered to borrowers who register themselves at the OPAC.

    images/2024R3_MAINT_BRWCAT_WELCOME_EMAILS_thumb.png

    Click image to enlarge

    To set up customised welcome emails for a specific borrower category:

    1. Navigate to Maintenance > Circulation > Borrower Categories
    2. Select a borrower category and switch to the Notice Settings tab
    3. Scroll down to the Welcome Email section
    4. Customise the Content, Attachment path, Join online content, and other settings as required.
    5. Click Save or Save & Close.

    Once these settings are saved, the customised welcome email will be sent to borrowers belonging to that category when a new borrower account is registered, or when the Send Welcome Email button is clicked in the edit borrower registration interface.

    If no welcome email content is specified for a borrower category, the system will default to sending the welcome email based on the content configured in Maintenance > General > HTML Emails & Slips Configurator > Communication, under the Welcome section.

    To send welcome emails, the Send welcome emails parameter must be set to Yes in Location Maintenance for the registration location. Default registration categories and locations may be set for online registration forms on the Settings tab of Borrower Join Online in Maintenance.

    Additional customisation of OPAC login text

    Some new fields have been added to the Login section of the OPAC HTML Configurator to allow staff to customise more text labels for OPAC login pages (login box, and pages that require login e.g. reservation placement or event registration).

    To set the messages for OPAC login, staff user should perform these actions:

    1. Navigate to Maintenance > OPAC & Enquiry > OPAC HTML configurator
    2. On the General tab, scroll to the Login section and click the header to expand it
    3. Set custom text labels as required for;
      • Log in text
      • Information text
      • Library account text
      • Online registration prompt
    4. Click Save or Save & Close.

    Once the labels are set and saved, they will be displayed at OPAC.

    Add privilege control to 'Delete Saved Query' action

    Previously, there was no user privilege level associated with the ability to delete a Saved Query. A new privilege option has been added to Privilege Manager to allow or disallow users access to this action. The Delete Saved Query privilege is listed in Privilege Manager under the Maintenance > Operations menu.

    The button text has also been changed from 'Delete Saved Query' to 'Delete' at Saved Query record Full Display.

    Based on the privileges set in Privilege Manager, users with sufficient privilege may delete

    • queries created by themselves
    • public queries,
    • queries created by other staff

    Users without sufficient privilege may only delete their own queries and public queries.

    Saved Queries being used by other features are restricted from deletion, including:

    • Reading Challenges
    • OAI-PMH
    • Widgets
    • Containers

    Saved Queries linked to Reading Challenges will not display the Delete button. A message will be displayed when staff attempt to delete OAI, widget or container queries.

    Allow customisation of home, join and login OPAC icons

    In the OPAC HTML Configurator, staff may now select an icon from the available FontAwesome icon pack to replace the default Home, Login and Join online icons.

    • The Home icon will be displayed on both desktop and mobile
    • The Login and Join online icons will be displayed on mobile

    To configure the icons:

    1. Navigate to Maintenance > OPAC & Enquiry > OPAC HTML Configurator.
    2. On the General tab, locate the Icon to use for Home link field (near the top of the page beneath the Home page URL) and select an icon from the picker.
    3. Scroll to and expand the Login section and select the Icon to use for Login link (mobile view).
    4. Scroll to and expand the Join Online/Update personal details section and select the Icon to use for Join online link (mobile view).
    5. Click Save or Save & Close.

    Customise OPAC error message for borrowers without a hoopla account

    This feature requires license version 11.7 or later.  

    It is required to have a hoopla account to borrow hoopla eResources at the OPAC, and an error message will be displayed if a borrower without a hoopla account attempts to access hoopa resources. Library staff can now customise this error message.

    The new When unable to create an account message in hoopla's section of the eContent Help tab in the OPAC HTML Configurator allows customisation of this error message.

    To set the message:

    1. Navigate to Maintenance > OPAC & Enquiry > OPAC HTML Configurator.
    2. Switch to the eContent Help tab and scroll to the hoopla section.
    3. Click the hoopla header to expand the section.
    4. Enter your message in the When unable to create account field.
    5. Click Save or Save & Close.

    Configure OPAC display of Work Instance records

    It is now possible to customise the Brief and Full Display of Work records at the OPAC in the new Work Instance section on the General tab of the OPAC HTML Configurator.

    Staff may customise various work instance elements related to the titles table, and Bibliographic record formats.

    1. Navigate to Maintenance > OPAC & Enquiry > OPAC HTML Configurator.
    2. On the General tab, scroll down to the Work Instance section and expand it.
    3. Set the Show titles on page load parameter
      • If enabled, the titles table will be expanded automatically open when the page loads, showing all title details
    4. Set the Show toggle link parameter
      • If enabled, a text link will be displayed at top of the titles table allowing OPAC visitors to click to expand/collapse the titles table
    5. Set Toggle Link text
      • If Show toggle link is available, the Toggle link text will be the text label of the link that allows OPAC visitors to click to expand/collapse the titles table
    6. Set the Show titles table header parameter
      •  If checked, the titles table header row will be displayed at top of the titles table
    7. In the Bibliographic record formats table, click the edit icon () at the end of any row to change the text Description of that bibliographic record format
      • The Description here will also be used in the Record format OPAC facet
    8. To reorder the display of the record format tabs at work, select a row and click Move Up or Move Down, or drag-and-drop into position.
      • The vertical (top-to-bottom) positioning in this list will match the horizontal (left-to-right) positioning of the record format tabs at the Brief & Full Display of Work records
    9. Click Save or Save & Close.

    Control which Locations appear in OPAC search lookups

    Staff can now customise which Locations appear in OPAC search lookups using a new parameter added to the Locations Maintenance.

    The Item location OPAC parameter allows staff to hide or unhide locations from the Location selection field in OPAC search forms.

    To set this parameter:  

    1. Navigate to Maintenance > Circulation > Locations
    2. Select the required location
    3. On the Properties tab, scroll to the General section.
    4. Set the Item location (OPAC) parameter
      • No - Hide location from search page Location lookups
      • Yes - Display location in search page Location lookups
    5. Click Save or Save & Close to apply changes
    NOTE: The Item location (OPAC) parameter and the OPAC suppression filter are separate functions. Ensure that if you wish to both suppress items from the Location from display and suppress the Location name from search lookups that both features are configured as required.

    Add option to enable/disable standing reservations in Spydus staff interface

    Staff can now control the display of the Place Standing Reservation button at the Full Display of a Serial in the Spydus staff apps. Previously this required commissioning to be performed by Civica staff.

    A new Enable standing reservation placement parameter has been added to the Reservation section of the General tab of the Enquiry HTML Configurator.

    Add control of global expansion in Search Parameters client

    This feature was introduced after being requested by Dublin City Libraries on the Spydus Ideas Portal (SPY-I-260). This idea received 30 votes!

    A new Enable global expansion setting has been added to Search Parameters Maintenance, enabling staff to customise form-specific search criteria for Enquiry independent of configured Global/Consortium settings in the Enquiry HTML Configurator.

    images/2024R3_MAINT_SEARCH_PARAMS_GLB_EXP_thumb.png

    Click image to enlarge

    Previously, settings at the Global/Consortium section applied to the all Enquiry forms. This feature empowers staff to configure search parameters with customisable controls, allowing them to disable global expansion for specific search forms while keeping it enabled for others. e.g. Many libraries in consortia prefer global searching for holdings, but to search locally for borrowers

    To configure this feature:

    1. Navigate to Maintenance > OPAC & Enquiry > Search Parameters 
    2. Select the required search Format from the dropdown list
    3. Set the Enable global expansion dropdown as required
      • Use Enquiry HTML Configurator settings - This search form will use the default global search settings as configured in Global/Consortium
      • No, local only - Local search results only; disables global expansion option for this search form
      • Yes, start with local search - Local institution results are displayed, with global search expansion available
      • Yes, start with global search - Shows results from multiple institutions, with option to filter to local catalogue search
    4. Click Save or Save & Close.
