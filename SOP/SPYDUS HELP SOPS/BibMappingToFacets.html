    Mapping of MARC tags
    In This Topic

    Spydus supports MARC21 bibliographic records. When a bibliographic record has been created from MARC21, the original MARC21 content is stored in the record and the data from the MARC tags is extracted into Spydus metadata (groups and elements) for indexing, retrieval and display purposes. Authority data from the bibliographic record is used to match to existing authority records or to create new authority records to which the bibliographic records are linked.

    Bibliographic facets are created using the authority data, coded data and other controlled data in the bibliographic records.

    Item records may represent physical items or digital item. Digital item are linked to Digital Asset records which contain additional metadata describing the digital asset and a link to the digital asset if it is stored externally to Spydus.


    MARC Bibliographic Mapping  

    • The original MARC content is loaded into the MARC group and is kept in its entirety.
    • Data is extracted from the MARC record and is loaded into Spydus groups in the BIB record format.
    • If the MARC record is updated, the existing data in the Spydus BIB record will be rebuilt using the changed MARC data.
    • Some of this data is used to link to existing authority records or to create new authority records, and these authority records are then linked to he BIB record.
    • Some data is stored as text, some as coded data, some as controlled data (such as dates), and some as links to other records (such as authority records).
      •  Some of this data is indexed and so is available for retrieval purposes and some of this data is used for display purposes.
    • The record displays are controlled using display parameters records which allow different data to be displayed in the staff and public interfaces. Some data not loaded into Spydus groups can be displayed directly from the MARC data if required.
    • The original complete MARC data can be displayed in the staff and public web interfaces.
    • Indexed data can be searched separately, or can be combined using Spydus search dictionary items. The Spydus Anywhere search is a combined search of many separately indexed text elements.

    Data from MARC tags is loaded into groups in the BIB format, as identified in columns one and two the following table. In some cases, data is only loaded depending on indicator settings. Each group contains one or more elements. The following table indicates the group – and from which MARC tags/offset position each group is populated.

    Please refer to the database configuration web page for details of each group, the elements within each group, the data type of each element and index information for each element.

    Facets in Spydus

    Search results pages in Spydus, whether brief or full display, will include facets to assist with refining search results. Facets are grouped into types such as format, audience, or publication date. Clicking on a facet link will refine the search page to display only those results from the original search that meet the additional criteria. Each facet shows the total number of records (from within the result set) that meet that facet's criteria.
