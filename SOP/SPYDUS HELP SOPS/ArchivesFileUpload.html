    File Upload for Archives
    In This Topic

    This utility is provided to allow libraries to upload digital files and the corresponding descriptive data into the Spydus Archives module on behalf of community groups. To be uploaded, the files must be accessible from a local folder, and the descriptive records provided in a (pre-defined) .CSV file.

    A template of this file can be downloaded by clicking the Download Template button in the action bar.

    File Upload

    To access the Archives File Upload utility;

    1. Navigate to the Archives module
    2. Expand the Other menu
    3. Select the File Upload option
    4. Select the Digital assets folder where files linked to the records will be loaded to
    5. Click the Select Files button
    6. Select all files to be uploaded and linked to the Archival Descriptions being loaded, then click Open
      • All files must be in the same local folder
    7. Click Select descriptive file
    8. Browse to the location of the .CSV file containing the descriptive information for the Archival Descriptions and click Open
      • See the File Upload Template heading below for more information on the correct format of the file
    9. To ensure that the .CSV formatting and files are correct before loading, click Validate in the action bar
      • Validation must be performed before Spydus will allow processing of the file
    10. Review the validation Output to see if any filenames or descriptive data requires editing
      • If any failures, fix the .CSV or selected files then repeat steps 4-9
    11. Click Process

    File Upload Template

    Only the provided .CSV template (ArcLoadTemplate.csv) or one with identical column structure may be used in the File Upload utility.

    Template rules

    • Each column in the .CSV template has a heading, and the headings of mandatory fields are marked with an asterisk
      • Users may not add columns to the .CSV file or use existing columns for different purposes (validation will fail) 
      • File formats other than .CSV are not supported, and the first row is assumed to be a header row
    • Each row in the .CSV will generate an Archival Description record of the Item level
    • Each record in the .CSV must link to an existing parent Archival Description record via the Parent ID column
      • The Parent ID field must match the Reference Code of the parent record
      • As the records created by File Upload are all loaded as Item level records, the parent should be a Series, Collection or Fonds level record
    • Each Archival Description record links to the file record in the File name column
      • Files can be viewed from the Archival Description record or in the nominated Digital Asset repository folder in the ERM module
    • Each file being loaded must match a row in the .CSV and vice versa
    • Files are uploaded to the nominated Digital Asset folder in ERM prior to the creation of the Archival Description record.
    • IDs may only contain alpha-numeric characters, or . - / _ * (all characters will be accepted in validation and loading, but some will be ignored by indexing and this may result in false duplicates during validation)
    • Where a Creator is provided in the descriptive data, a Name record is created (or linked if the name already exists) as well as a heading. See diagram below for the logic of creating/linking Name and Authority records
    • Authority records are created from the Subject/s. Multiple Subjects can be accommodated in a single record if they are pipe separated (e.g. History | Australian history)
    • In addition to the columns of data available in the .csv file, defaults are applied as the records are created. See the Default values heading below for more information

    Default values

    The following deafult values are applied to all Archival Description records created via File Upload:

    • Archival description category = 5 / Item
    • Archival description level = 50 / Item
    • Item details type = I / Single item
    • Item details copy = 1
    • Item details digital = 1
    • Item details may be issued = 0
    • Item details may be reproduced = 0
    • Encoding level = full
    • Supplied title flag = 0
    • Status = Finalised
    • Suppress from OPAC = 0
    • Default type (GMT) = GM / Graphic Material
    • Creator heading category = N / Name
    • Creator heading type = PN1 /
    • Creator heading record category = NS / Personal Name – Surname
    • Heading status = F / Finalised
    • Heading encoding level = P / Partial
    • Preferred heading flag = 1
    • Verified heading flag = 0
    • Orphan heading flag = 0
    • Thesaurus code = Local
    • Subject heading category = S / Subject
    • Subject heading type = TT / Topical Term
    • Name type = P / Personal
    • Name status = D / Draft
    • Name encoding = M / Minimal

    It is possible that future versions of this utility will accommodate more columns in the .csv file, including the defaults above.
