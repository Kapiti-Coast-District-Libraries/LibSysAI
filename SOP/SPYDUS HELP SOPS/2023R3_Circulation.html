    Ability to anonymise resigned borrower

    This feature requires server version 10.9 or higher.  

    It is now possible to anonymise borrowers automatically after a nominated period following borrower resignation. If configured, this means that even borrowers who did not specifically request anonymisation will still be anonymised.

    If the Anonymisation period for resigned borrowers has elapsed - that is, that a borrower was resigned longer ago than the nominated period (in days) - then the borrower anonymisation scheduled task will anonymise that borrower.

    Example

    The Anonymisation period for resigned borrowers is set to 365 days. John Smith logs into his account at the OPAC, and requests to cancel his membership, but does not choose the 'permanently forget my membership' option. John's account is deactivated, but his details are not flagged to be anonymised.

    A year later, during which time John has not reactivated his account, John's details will be anonymised at the next run of the anonymisation scheduled task.

    The Anonymisation period for resigned borrowers parameter is located in Maintenance > Circulation > Global > General Parameters.

    For libraries on Spydus 10.9, please contact Civica Support for assistance as commissioning is required.

    Amend base URL used for Addressify API endpoints in Circulation

    This feature is available to all libraries on; 

    • 2023R3 (no license requirement)
    • W10.9G (no server requirement)

    In Circulation, the base URL for Addressify API endpoints has been updated from:

    • https://addressifyapiau.azurewebsites.net/

    ..to the preferred:

    • https://api.addressify.com.au/

    Prompt to print transactional slips if enabled 

    If Slip Printing Parameters are configured to do so, Spydus will prompt the user to optionally print a copy of a transactional slip.

    In order for the prompt to display, the borrower for whom the slip is being generated must have their slip printing preferences set to an option that includes printing (e.g. E-Mail and Print). Any other selected slip delivery methods will be generated, but the prompt will allow staff to determine whether a paper slip is printed.

    This feature was introduced after being requested by Luisa at Yarra Plenty Regional Libraries on the Spydus Ideas Portal (SPY-I-806). This idea received 34 votes!
