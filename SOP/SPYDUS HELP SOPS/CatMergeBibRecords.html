    Merge bibliographic records
    In This Topic

    This feature requires server version 10.5.1 or higher.  

    To merge two bibliographic records:

    1. Navigate to the Cataloguing module.
    2. Perform a Bibliographic Search to locate the record that you wish to merge from (i.e. the record that will be lost in the merge). Open the record Full Display and select Edit Record from the action bar.
    3. In the Edit Record display, click the Tools drop down menu and select Merge Record.
    4. This will produce a Bibliographic Search popup. Perform a search for the record you will be merging into (i.e. the record that will be kept and contain all holdings after the merge). Click through to the Full Display of the record, and click Select Record.
    5. Click to select the load control to be used to perform the merge, then click the Select button at the bottom of the pop-up window.
      The parameters of the selected load control will determine which tags from each record are retained and which are overwritten. See the Merge Bibliographic Load Controls section below for more detail.
    6. Spydus will present a prompt: 

      Click Yes to proceed with the merge.

    After merging, Spydus will be proceed to the new merged record (displaying all holdings, loans, reservations, orders, etc from both bibliographic records).

    Merge Bibliographic Load Controls

    In all cases, the BRN/IRN of the to record will be retained, however, the load control selected to perform the merge allows the user to determine which tags from each record are retained or replaced.

    When prompted to select the load control (step 5 above), the user will be presented with an interface similar to the Load Control Maintenance UI. However, these Load Controls have limited parameters, and are used exclusively for merging records.

    Replace existing record entirely

    Selecting this option will replace all existing tags in the to record with all existing tags in the from record, as well as adding any tags in the from record that do not exist. 

    Retain existing record but load some incoming tags

    Selecting this option will retain all tags in the to record, except for those specified in the Tags wanted... section, which will be created where they do not exist, or for repeatable tags, the existing tag(s) will be retained and the new one(s) will be loaded.

    Replace existing record but retain some tags

    Selecting this option will replace all existing tags in the to record with all existing tags in the from record, except for those specified in the Tags wanted... section, which will be retained. For repeatable tags, the existing tag(s) in the to record will be retained, and the new one(s) will be loaded.

    Retain existing record but load incoming tags if it does not exist

    Selecting this option will retain all tags in the to record, and load any tags in the from record that do not currently exist.
