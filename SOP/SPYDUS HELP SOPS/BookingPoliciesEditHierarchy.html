    Edit policy hierarchy
    In This Topic

    The overriding policies are specified in a policy "hierarchy".

    When you try to place a booking or place a reservation or issue an item or generate a notice, Spydus will work through the appropriate hierarchy (for booking, reservations, loans or notices) searching for an overriding policy to be applied. If there is no overriding policy then the default policy will be applied.

    Set up the common parameters in the default policy, and then set up any overriding policies you require. You can set up as many overriding policies as you wish, but the golden rule is keep it simple.

    As a general rule, overriding policies should be listed according to the number of components in the scope. For example, policies with three components (e.g. location, borrower category and collection) would usually come before those with two (e.g. borrower category and collection) which would usually come before those with one (e.g. collection).

    To set up or edit a policy hierarchy:

    1. From the Module Menu select Maintenance, Circulation, Policies and then the policy you want — Booking Policies or Global Reservation Policies or Loan Policies or Notice Policies or Reservation Policies.
    2. Click Edit Hierarchy.
    3. Do one or more of the following.
      • To add a new scope to the hierarchy, select one of the Available policies and click Add.
      • To remove a scope from the hierarchy, select one of the Applied policies and click Remove.
      • To move a scope up or down in the hierarchy, select one of the Applied policies and click Up or Down.
    4. Click Save or Save & Close.
